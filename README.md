# HA KiSa Uni Plan

Универсальный менеджер последовательных сценариев для Home Assistant. 

Позволяет создавать настраиваемые цепочки действий (включение света, запуск кофеварки, открытие штор, запуск робота-пылесоса) с заданными задержками между ними. Интеграция дает полный UI-контроль над порядком выполнения, расписанием по рабочим дням и позволяет в один клик отключать как всю цепочку целиком, так и отдельные ее шаги прямо с дашборда.

## Особенности

* **Полный контроль через интерфейс (UI):** Нет необходимости писать сложные YAML-автоматизации. Добавляйте, удаляйте, меняйте порядок и задержки шагов прямо в настройках интеграции.
* **Дочерние выключатели (Child Switches):** Для каждого шага плана автоматически создается отдельный выключатель. Вынесите их на карточку в дашборде, чтобы быстро отключать ненужные шаги (например, отключить включение света летом), не прерывая выполнение остального плана.
* **Умное расписание:** Запуск по заданному времени с возможностью выполнения только по рабочим дням. Для этого используется интеграция [Workday](https://www.home-assistant.io/integrations/workday/) (встроенная интеграция Home Assistant, которая определяет рабочие, выходные и праздничные дни). Ее необходимо добавить в Home Assistant отдельно перед использованием этой функции.
* **Безопасное выполнение:** Если вы выключите главный свитч плана во время его выполнения (даже во время длительной паузы), автоматизация немедленно прервется.
* **Сохранение состояния:** Компонент "помнит" состояние планов и шагов (включено/выключено) даже после перезагрузки Home Assistant.

## Пример использования: "Подъем на работу"

Вместо сложной автоматизации с условиями, вы можете просто создать **План** в интерфейсе со следующими шагами:
1. Запуск чайника (задержка: 0 сек).
2. Включение подсветки на кухне (задержка: 300 сек - включится через 5 минут).

Все сущности будут доступны в виде выключателей на вашей карточке:
* Главный выключатель `switch.podem_na_rabotu` (активирует или деактивирует весь план).
* Выключатель первого шага: `switch.podem_na_rabotu_step_0_kettle` (Шаг 1 - Чайник).
* Выключатель второго шага: `switch.podem_na_rabotu_step_1_light` (Шаг 2 - Подсветка).

Если летом рано светлеет, вы можете просто выключить переключатель второго шага (Шаг 2 - Подсветка) прямо на карточке, и этот шаг будет пропущен, но чайник все равно включится!

## Установка через HACS

**Внимание:** Если вы планируете использовать расписание по рабочим дням, предварительно убедитесь, что у вас установлена и настроена интеграция [Workday](https://www.home-assistant.io/integrations/workday/).

1. Откройте **HACS** в вашем Home Assistant.
2. Перейдите в раздел **Интеграции** (Integrations).
3. Нажмите на три точки в правом верхнем углу и выберите **Пользовательские репозитории** (Custom repositories).
4. Вставьте ссылку на этот репозиторий: `https://github.com/kornalexandr2/ha-kisa-uni-plan`
5. Выберите категорию **Интеграция** (Integration) и нажмите **Добавить** (Add).
6. Найдите добавленный **HA KiSa Uni Plan** в списке пользовательских репозиториев HACS и нажмите на него.
7. В правом нижнем углу нажмите кнопку **Скачать** (Download).
8. Перезагрузите Home Assistant, если система об этом попросит (обычно это уведомление появляется в Настройках).

## Настройка

1. Перейдите в **Настройки** -> **Устройства и службы**.
2. Нажмите **Добавить интеграцию** (Add Integration) и найдите **HA KiSa Uni Plan**.
3. Укажите название плана, время старта и опционально сенсор рабочих дней.
4. После создания плана, найдите его в списке добавленных интеграций и нажмите кнопку **Настроить** (Configure), чтобы добавить шаги: укажите сущность, действие (включить/выключить/переключить) и задержку (в секундах) перед выполнением этого действия.
5. Радуйтесь удобным автоматизациям!
